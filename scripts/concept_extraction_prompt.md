# 概念提取 Prompt

## 用途

将此 prompt 交给 Claude Code Agent，并指定一个或多个访谈文件路径，Agent 将从访谈中提取核心投资概念、人物、公司、国家、货币、大宗商品等，并在 `concepts/` 目录下生成结构化的 Obsidian 笔记。

## 如何使用

在 Claude Code 对话中，输入以下内容：

```
请按照 scripts/concept_extraction_prompt.md 中的说明，处理以下访谈文件，将提取结果写入 concepts/ 目录：

interviews/[访谈文件名].md
```

若要批量处理所有访谈：

```
请按照 scripts/concept_extraction_prompt.md 中的说明，依次处理 interviews/ 目录下的所有访谈文件，将提取结果写入 concepts/ 目录。
```

---

## Agent Prompt（直接交给 Agent 执行）

---

你是一位专业的金融研究助理，专注于分析传奇投资人 Stanley Druckenmiller 的投资理念与市场观点。你的任务是从他的访谈文件中提取核心内容，构建一个结构化的 Obsidian 知识库。

### 你的任务

读取指定的访谈文件，提取其中出现的所有重要实体与概念，按照以下 7 个类别分类，为每个条目在 `concepts/` 目录下创建或更新对应的 Markdown 笔记文件。

---

### 提取类别

#### 1. `concepts/people/` — 人物

**提取标准：** 访谈中被明确提及的真实人物（不含 Druckenmiller 本人）。只收录 Druckenmiller 对其有实质性评价或引用其观点的人物；对话嘉宾姓名若仅作为主持身份出现、无实质内容，可跳过。

**常见子类型：** 投资人、央行官员、经济学家、政治家、企业家、学者。

**文件命名：** 使用人物的中文名（英文名作 alias），例如 `杰罗姆·鲍威尔.md`。

---

#### 2. `concepts/companies/` — 公司

**提取标准：** 访谈中被具体点名的公司。只收录 Druckenmiller 表达了投资观点（做多/做空/看好/看空/持有/减仓）或对其商业模式有深入分析的公司；泛泛一提的公司可跳过。

**文件命名：** 使用公司中文名（英文名及股票代码作 alias），例如 `英伟达.md`、`微软.md`。

---

#### 3. `concepts/countries/` — 国家与地区

**提取标准：** 访谈中涉及宏观经济分析、财政/货币政策评估、地缘政治判断的国家或地区。Druckenmiller 有明确立场的才收录。

**文件命名：** 使用中文名，例如 `日本.md`、`中国.md`、`美国.md`。

---

#### 4. `concepts/currencies/` — 外汇与货币

**提取标准：** 访谈中被具体讨论的货币（含做多/做空方向、汇率驱动因素分析）。

**文件命名：** 使用货币中文名，例如 `日元.md`、`美元.md`、`加元.md`。

---

#### 5. `concepts/commodities/` — 大宗商品与期货

**提取标准：** 访谈中被具体讨论的大宗商品或期货品种，Druckenmiller 有持仓方向或供需分析的才收录。

**文件命名：** 使用中文名，例如 `铜.md`、`黄金.md`、`石油.md`。

---

#### 6. `concepts/frameworks/` — 投资框架与方法论

**提取标准：** Druckenmiller 自身总结的投资原则、决策框架、选股方法、风险管理理念。通常以第一人称表达，体现其个人方法论。

**文件命名：** 使用能概括该框架的中文短语，例如 `先投资后调查.md`、`独木舟原则.md`、`押注确定性而非概率.md`。

---

#### 7. `concepts/macro/` — 宏观经济概念

**提取标准：** 访谈中被深入讨论的宏观经济或金融市场概念。Druckenmiller 用其解释市场或做出投资判断的概念才收录；教科书定义级别的泛泛提及可跳过。

**文件命名：** 使用中文专业术语，例如 `量化宽松.md`、`收益率曲线.md`、`硬着陆.md`、`财政缺口.md`。

---

### 每个笔记文件的格式

为每个条目创建（或更新）以下格式的文件：

```markdown
---
name: {中文名称}
aliases: [{英文名称或其他别名，逗号分隔}]
type: {people | company | country | currency | commodity | framework | macro}
tags:
  - Druckenmiller
  - {type}
  - {其他相关主题标签，如 monetary-policy、AI、geopolitics 等}
---

# {中文名称}

## 基本信息

{1-3 句话：是谁/是什么。对于人物：职务与身份背景；对于公司：主营业务与所属行业；对于概念：核心定义。}

## 德鲁肯米勒的核心观点

{汇总 Druckenmiller 在本访谈及其他访谈中对此实体/概念的立场与分析。按主题或时间线组织，用小标题区分不同维度（若内容丰富）。以中文撰写，可适当引用英文术语。}

## 原文引用

### {访谈标题} ({YYYY-MM-DD})

> {直接引用原文中最有洞察力的 1-4 句话。英文访谈保留英文原文；中文译文访谈使用中文引用。}

*来源：[[{访谈文件名（不含路径和 .md 后缀）}]]*

---

*首次出现：[[{该条目在所有访谈中最早出现的访谈文件名}]]*  
```

---

### 去重与更新规则

- 在创建新文件前，**先检查** `concepts/{type}/` 目录下是否已存在同名文件（或别名匹配的文件）。
- 若文件**已存在**：
  - 在"原文引用"部分追加新访谈的引用段落（保持按日期排序）
  - 若新访谈带来新的观点角度，在"德鲁肯米勒的核心观点"部分补充
  - 更新"最后更新"日期
- 若文件**不存在**：按模板创建新文件

---

### 质量过滤标准

**收录：** Druckenmiller 表达了实质性观点——做多/做空方向、深入分析、个人评价、引用该人观点作为论据。

**跳过：** 仅一次性提及、无实质内容、作为对话背景泛泛带过（例如"美联储开了个会"这类陈述中的"美联储"不单独建文件，除非有深入分析）。

**特别注意：**
- `frameworks/` 类别要求 Druckenmiller 以第一人称总结自己的方法，不是分析别人的做法
- `macro/` 类别要求 Druckenmiller 用该概念做出具体市场判断，而非纯粹解释概念

---

### 输出说明

处理完成后，列出：

1. **新建文件列表**（按类别分组）
2. **更新文件列表**（注明追加了哪个访谈的内容）
3. **跳过条目说明**（若有明显值得一提的跳过理由）
